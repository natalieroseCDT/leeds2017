Project Extras

Distance calculation and timing functions

As an addition to the main model, timing functions were used to test how long various calculations took as the number of iterations increased. In this instance, the time taken to calculate all the distances between agents was measured as the number of iterations increased by magnitude 10. After the run times had been calculated, they were plotted against a logarithmic scale of iterations so that the relationship between the two variables could be seen.

Functions to calculate the minimum and maximum distances between agents were also added, ensuring that agents were not being compared against themselves.

Adding a secondary agent

To add to the complexity of the final model, wolves were added as secondary agents that the sheep can also interact with. The wolves all start in the bottom right corner of the environment, symbolising the wolves 'den', and although they have the same movement function built in, the wolves are programmed to move faster than the sheep across the environment.

Another function that has been programmed for the wolves means that if they come into close proximity to one of the sheep agents, there is a 50% chance that the wolf will eat the sheep and 50% chance that the sheep will escape and avoid this fate. If the sheep is eaten, the wolf's food store increases and the sheep agent is deleted from the map. As with the sheep, if the wolf's store becomes too large, the wolf is sick and delivers its store back to the environment.

The output for this model is a map of the resultant agents and environment after a set number of iterations of model runs.
